Goals for my language:

- It should be a simple and intuitive language
  - I want people to read the code and just understand what is going on
  - No abbreviations, no cryptic macros
  - A cleanly named standard library
- There should be only one way to solve a problem
  - I want the language to provide only as little syntax and as features as needed to solve problems
- It should be explicit
  - The programmer should always have a clear understand of what the specific line of code they are writing is actually doing
    - In terms of memory, and performance
- I want the language to be expressive
  - Despite its simplicity and explicitness, I want people to express complex ideas with as litte code as possible and as much as necessary
  - It should not be verbose
  - Solving and performing everyday tasks should not feel like a ceremony

- Refactoring should be easy
  - Renaming a variable or changing a type should immediately tell you where stuff does not work so you can fix it

- Tooling
    - Simple project structure
    - Easy definition of modules
    - Easy imports of other modules
    - Good error messages
    - Effortless to set up
    - Sane defaults
    - As little configuration as possible
